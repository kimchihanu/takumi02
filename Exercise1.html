1章 Firebugを利用して通信内容を見てみる

1.1 Firebugのセットアップ

これからブラウザがどのように通信しているかを調べるため、FirefoxとFirebugをインストールする。
Firefox Setup 13.0.1.exeを実行してインストールする。
インストールが終わったらFirefoxを起動し、firebug-1.9.2-fx.xpiをFirefoxのウィンドウにドラッグアンドドロップする。 FirebugはF12キーを押すと表示される。
今回はFirefoxの通信内容を見たいのでネットタブを押下する。

1.2 通信の内容を表示する

  • アドレスバーに下記URLを入力してEnterキーを押す。
    • ・文部科学省 http://www.mext.go.jp/
    大量の通信が実行されているのがわかる。
  • www.mext.go.jp のヘッダをクリックする。
  • レスポンスヘッダとリクエストヘッダの「ソースを表示」をクリックする。

    • 1.3 ここまでのまとめ

    ここまで何が起きているかを順を追って理解する。

    1. ユーザがURL（http://www.mext.go.jp/）を入力してエンターキーを押す。
    2. ブラウザはWEBサーバに対してリクエストを送信する。
    3. WEBサーバはブラウザに対してレスポンスを送信する。
    4. ブラウザは受信したhtmlファイルを表示する。
    1.4 教室内のWEBサーバとの通信
    インターネットのサイトでのやり取りを見ると通信が複雑なので、ここからは、教室内でWEBサーバを起動してやり取りを確認する。
  • まずは、講師のWEBサーバにアクセスする。
  • ブラウザに下記を入力する。
    • http://XXX.XXX.XXX.XXX/
    ※XXXの部分は講師に指定されたのもを指定する。
  • 下記ページが表示される。
    • XXX.XXX.XXX.XXXで表現される文字列をIPアドレスという。 IPアドレスは通信機器に付与するアドレスである。通常はOSに設定する。Windowsの場合はコマンドプロンプトで「ipconfig」というコマンドを実行すると、 設定されたIPアドレスをアドレスを確認できる。
  • 次に受講生のマシンにWEBサーバをインストールし、受講生がWEBサーバを公開してアクセスしてみる。

    • 1.5 WEBサーバのインストールと起動
  • ここではanhttpdという簡易的なWEBサーバを使う。
  • httpd142p.exeを実行する。
  • ダイアログにはc:\anhttpdと入力してUnzipボタンを押下する。
  • C:\anhttpd\httpd.exeを実行する。

    • 1.6 WEBサーバを設定する

  • タスクトレイのhttpdをクリックする。
  • [オプション]-[一般]をクリックする。
  • ドキュメントルートに「c:\anhttpd\docroot」と入力してOKボタンを押下する。

    • 1.7 公開するページを作成する
    c:\anhttpd\docroot\index.htmlを下記内容で作成する。
    <html>
    <body>
    hello my name is [あなたの名前]
    </body>
    </html>
